Summary

Andrew Shen, Charles Liu, Lena Kushigemachi, Ryan Chan

In this project, we will implement a first-person shooter game inspired by Counter-Strike, a popular FPS video game series. The focus of our project will be on interactive volumetric smoke grenades along with implementing functionality to make the game interactive and playable with real-world physics.

Problem Description

Problem:

Our project will recreate the basic functionality of a popular first-person shooter (FPS) game, Counter-Strike. We want to specifically focus on creating realistic volumetric smoke grenades that will interact with the environment. This includes bullet tracers and environmental structures. This is inspired by the recent announcement of Counter-Strike 2 and its introduction of responsive smokes, dramatically changing gameplay from its predecessor, Counter-Strike Global Offensive. Counter-Strike has provided entertainment and enjoyment for millions of players and we want to figure out the specific mechanics that have contributed to its success. We want to create a playable game with player movement and shooting physics.

Challenges:

Creating a game from scratch is complex and involves a lot of different aspects that will need to work cohesively together, such as kinetics, cameras, lighting, and texturing. Combining the different aspects of the game together could produce unexpected bugs that only appear when run altogether, making it difficult to debug. In addition, creating interactive smokes that can respond to the environment will be challenging.

Solution:

Using Godot as a game engine, we will create an environment and implement player physics and collisions. We will also implement interactive smokes using volume fog based on volumetric lighting in a large 3-dimensional buffer, also known as froxel buffers. We will utilize kinematics to implement player movement and raycasts to implement bullet physics by following Godot documentation and tutorials. We will use keyframing to create animations.

Goals & Deliverables

What we plan to deliver:

  • Interactive demo:
    • Player movement
    • Shooting and bullet physics
    • Interactive smoke grenades
  • Videos of gameplay showing various aspects
  • Images of smokes and interaction with smokes

What we hope to deliver:

We hope to deliver more features to extend our project past the basics. These include more maps/different environments, multiplayer, custom skins, and different types of weapons. We want to make CS:GODOT feel more complete as a game with replayability and variety.

Measurements of the quality/performace of the system:

Player movement should be semi-realistic. Smokes should work as intended and produce a volumetric fog effect with the ability to physically alter the shape of the smoke. Raycasting and bullet tracing should work as intended. Bullets should follow the expected trajectory of the aimed path. Player animations should be seamless with no artifacts. Overall performance should be smooth with no lagging. We can analyze the affect of our program on our personal computers.

Schedule

Week 1:

  • Learn about different algorithms and techniques to model volumetric smoke in different environments
  • Learn how to use Godot in either the C++ variant or the GDScript variant
  • Set up project and figure out how to enable efficient cooperation
  • Determine what sort of resources we can use to find good assets for our world or how to use tools in order to create such resources
  • Begin coding basic player movement and collision detection

Week 2:

  • Begin coding volumetric smokes using aforementioned algorithms and techniques to model it properly
  • Begin coding smoke grenades physics
  • Begin coding weapon physics
  • Determine the resources/assets that we need for grenade models, gun models, etc.
  • Plan layout and aesthetic of the map as well as being to insert the correct assets into the game

Week 3:

  • Continue coding volumetric smoke based on techniques and algorithms as described above
  • Continue to work on expanding the map and focus on adding details that bring life to the scenes

Week 4:

  • Perform bug testing and examine edge cases to make sure that the code runs as expected
  • Put finishing touches on the map and improve the visual quality by adding lighting, wind, and shaders
  • Prepare presentation and deliverables
